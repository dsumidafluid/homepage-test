Windows Organizing

Windows データ整理術

片付けが苦手でも続けやすいように、探す時間を減らすためのフォルダ、ファイル名、一時置き場、検索、バックアップのルールをまとめます。

Rules

まず決める基本ルール

保存場所を増やしすぎない

デスクトップ、ダウンロード、ドキュメント、OneDriveなどに散らばると探せなくなります。主戦場を1つ決めます。

例:
D:\work
D:\private
D:\archive

一時置き場を作る

すぐ分類できないものは一時置き場へ入れます。散らかりを許す場所を1つに限定します。

00_inbox
00_temp
00_sort_later

週1回だけ片付ける

毎回完璧に分類しようとすると続きません。週1回、15分だけ一時置き場を空にします。

金曜 16:30
00_inboxを空にする

迷ったら消さずにArchive

削除に迷うと片付けが止まります。迷うものは年月付きのアーカイブへ移します。

archive\2026\2026-06_old_files

Folders

探しやすいフォルダ構成

番号で並びを固定

よく使う順番に番号を付けると、名前順でも崩れません。

00_inbox
10_projects
20_reference
30_output
90_archive

案件フォルダは型を作る

毎回同じ構成にすると、どこに何を置くか迷いません。

project_name
  01_input
  02_work
  03_output
  04_memo
  90_archive

資料と成果物を分ける

受け取った資料、作業中ファイル、提出物を混ぜないようにします。

input: 受領物
work: 編集中
output: 提出物

デスクトップは作業台

長期保存場所にせず、今日使うものだけ置きます。終業時に空にするのが理想です。

Desktop:
今日使うものだけ

Names

ファイル名ルール

日付を先頭に付ける

日付順に並び、最新版が分かりやすくなります。

2026-06-29_meeting-note.md
2026-06-29_report_v01.xlsx

版数を付ける

final、final2、最新版のような名前を避けます。

proposal_v01.pptx
proposal_v02.pptx
proposal_v03_submit.pptx

検索語を入れる

後で探す言葉をファイル名に入れます。相手名、案件名、種類、日付が有効です。

2026-06-29_A社_見積_v01.xlsx

記号を少なくする

環境差で困りやすい記号を避け、ハイフンやアンダースコア程度にします。

OK:
2026-06-29_project_memo.txt

Cleanup

片付け下手向けのお助けテクニック

ダウンロードを毎週空にする

ダウンロードは仮置き場です。必要なものは移動、不要なものは削除します。

Downloads:
必要 → 正式フォルダへ
不要 → 削除
不明 → 00_inboxへ

検索を前提にする

完璧な分類より、名前と場所を決めて検索できる状態を優先します。

検索語:
案件名
日付
相手名
種類

大容量ファイルを定期確認

動画、ZIP、古いインストーラー、重複バックアップが容量を使いがちです。

見る場所:
Downloads
Videos
Desktop
old_backup

バックアップを1つ持つ

大事なデータはPC内だけに置かず、外付けドライブやクラウドにも保存します。

最低限:
作業PC
外付け/クラウド
重要ファイルだけでも二重化

Storage Senseを使う

Windowsのストレージ管理機能で、一時ファイルや不要ファイルの整理を補助できます。削除対象は確認してから実行します。

設定 → システム → ストレージ

公式情報へのリンク

Windowsの検索、ストレージ、OneDriveなどの詳細はMicrosoft Supportを参照します。このページは整理の習慣メモです。

https://support.microsoft.com/windows
